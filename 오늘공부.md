1. @Controller vs @RestController
@Controller: "뷰(HTML)"를 찾아서 돌려준다. (데이터는 Model에 담아서)

@RestController: "데이터(JSON)" 그 자체를 돌려준다. (API용, @Controller + @ResponseBody)

2. @GetMapping vs @PostMapping
@GetMapping: "데이터 줘" (조회용, R). 데이터가 URL에 실림.

@PostMapping: "데이터 먹어" (생성/제출용, C/U). 데이터가 Body에 숨겨짐.

3. '웹뷰(클라이언트)'가 JSON 받는 법 (CSR)
서버가 @RestController로 JSON 쏨.

클라이언트(웹뷰)의 **'자바스크립트(JavaScript)'**가 fetch API를 써서 JSON을 '받아옴'.

받아온 데이터로 HTML 요소를 '수동으로' 조작해서 화면을 그림. (document.getElementById(...))

4. 깃헙(Github) 기본
.md: 마크다운 파일. 개발자용 '예쁜' 메모장.
